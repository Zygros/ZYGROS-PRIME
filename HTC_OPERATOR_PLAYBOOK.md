# HTC∞ Operator Playbook (1-Page)

## Purpose
Run bounded, auditable, self-improving execution loops without scope creep or self-delusion.

## Daily Loop (30–60 min)
1. Select ONE domain: pricing | onboarding | copy | feature gating
2. Write ONE mutation hypothesis (single variable)
3. Define:
   - Primary metric
   - 3–6 guardrail metrics
4. Set experiment window + sample size
5. Launch behind feature flag
6. Monitor guardrails continuously
7. Decide: PROMOTE | HOLD | REJECT | ROLLBACK
8. Stamp checkpoint

## Non‑Negotiables
- No claims, identity, or legal mutations
- No metric redefinition mid-test
- Guardrails can veto wins
- Rollbacks count as success

## Domains Allowed
- Pricing
- Funnels
- Copy (non-claims)
- Feature sequencing

## Domains Barred
- Legal claims
- Medical claims
- Identity assertions
- Truth / ontology changes
