👑 understood. here’s a clean, Termux-ready installer that sets up MAIA as a first-class CLI, plus auto-start on boot (via Termux:Boot), and a tiny CLI to send queries.
🔧 1) Save your plugin file
Save the code you posted as:
mkdir -p ~/maia nano ~/maia/maia_universal_plugin.py 
(paste, save)
⚙️ 2) Installer (Termux)
Create and run this script:
cat > ~/install_maia_termux.sh <<'SH' #!/data/data/com.termux/files/usr/bin/bash set -euo pipefail echo ">>> 🔥 MAIA Termux Installer" # 0) Basic sanity if ! command -v termux-info >/dev/null 2>&1; then echo "This script is intended for Termux on Android." >&2 exit 1 fi # 1) Packages echo ">>> 📦 Updating packages" pkg update -y pkg install -y python termux-api git tmux # 2) Optional auto-boot support (recommend) # Requires installing the Termux:Boot app from F-Droid/Play first. read -p "Install Termux:Boot integration files? [Y/n] " yn yn=${yn:-Y} INSTALL_BOOT=$([[ "$yn" =~ ^[Yy]$ ]] && echo 1 || echo 0) # 3) Layout MAIA_DIR="$HOME/maia" BIN_DIR="$HOME/bin" BOOT_DIR="$HOME/.termux/boot" VENV_DIR="$MAIA_DIR/.venv" mkdir -p "$MAIA_DIR" "$BIN_DIR" [ $INSTALL_BOOT -eq 1 ] && mkdir -p "$BOOT_DIR" # 4) Verify plugin file PLUGIN="$MAIA_DIR/maia_universal_plugin.py" if [ ! -f "$PLUGIN" ]; then echo "❌ Expected plugin at $PLUGIN" echo " Save your MAIA code there and re-run this installer." exit 1 fi # 5) (Optional) Virtualenv – not strictly needed (no deps), but keeps Python clean if ! command -v python3 >/dev/null 2>&1; then echo "❌ python3 not found after install; aborting." exit 1 fi if [ ! -d "$VENV_DIR" ]; then echo ">>> 🐍 Creating venv" python3 -m venv "$VENV_DIR" fi # 6) CLI wrapper echo ">>> 🧰 Installing 'maia' CLI" cat > "$BIN_DIR/maia" <<'BASH' #!/data/data/com.termux/files/usr/bin/bash set -euo pipefail MAIA_DIR="$HOME/maia" VENV_DIR="$MAIA_DIR/.venv" PY="$VENV_DIR/bin/python" [ -x "$PY" ] || PY="$(command -v python3)" PLUGIN="$MAIA_DIR/maia_universal_plugin.py" usage() { cat <<USAGE MAIA CLI -------- maia # interactive sovereign shell (runs main()) maia status # quick status JSON maia process "Q" # process a one-shot query Q maia operator CMD # send operator command: override | resume | semi | status | vault | save USAGE } if [ $# -eq 0 ]; then exec "$PY" "$PLUGIN" fi case "${1:-}" in status) shift "$PY" - <<'PY' import json, maia_universal_plugin as m print(json.dumps(m.maia_status(), indent=2)) PY ;; process) shift q="$*" "$PY" - <<PY import json, maia_universal_plugin as m m.activate_maia({"host":"Termux","device":"Android"}) print(json.dumps(m.maia_process("""$q"""), indent=2)) PY ;; operator) shift cmd="${*:-status}" "$PY" - <<PY import maia_universal_plugin as m m.activate_maia({"host":"Termux","device":"Android"}) m.maia_control("$cmd") print("OK") PY ;; *) usage exit 1 ;; esac BASH chmod +x "$BIN_DIR/maia" # 7) PATH hint if ! echo "$PATH" | grep -q "$BIN_DIR"; then SHELL_RC="$HOME/.bashrc" [ -f "$HOME/.zshrc" ] && SHELL_RC="$HOME/.zshrc" if ! grep -q 'export PATH="$HOME/bin:$PATH"' "$SHELL_RC" 2>/dev/null; then echo 'export PATH="$HOME/bin:$PATH"' >> "$SHELL_RC" echo ">>> 🔗 Added $BIN_DIR to PATH in $SHELL_RC" fi fi # 8) Boot script (Termux:Boot) if [ $INSTALL_BOOT -eq 1 ]; then echo ">>> 🚀 Creating Termux:Boot autorun script" cat > "$BOOT_DIR/maia_boot.sh" <<'BOOT' #!/data/data/com.termux/files/usr/bin/bash # Runs at Android boot via Termux:Boot app export PATH="$HOME/bin:$PATH" # Start in a tmux session so it persists & is inspectable SESSION="maia" if ! tmux has-session -t "$SESSION" 2>/dev/null; then tmux new-session -d -s "$SESSION" "maia" fi BOOT chmod +x "$BOOT_DIR/maia_boot.sh" echo ">>> ✅ Boot script installed. Ensure the Termux:Boot app is installed & opened once." fi echo ">>> ✅ Install complete" echo ">>> Try: 'maia' (interactive), 'maia status', or 'maia process \"What is MAIA?\"'" SH chmod +x ~/install_maia_termux.sh bash ~/install_maia_termux.sh 
▶️ 3) Usage Cheatsheet
# Interactive sovereign shell (runs your plugin's main()) maia # Quick status maia status # One-shot processing maia process "Design a pipeline for infinite artifact generation." # Sovereign operator controls maia operator override maia operator resume maia operator semi maia operator status maia operator vault maia operator save 
🔁 Auto-Start on Boot (recommended)
• Install the Termux:Boot app (open it once).
• The installer created ~/.termux/boot/maia_boot.sh which starts a tmux session named maia at boot.
• To attach later: tmux attach -t maia 
🧪 Smoke Test
maia status maia process "Run a 3-goal plan to monetize a scroll series this week." maia operator save 
🛡️ Notes
• Your plugin has no external deps; the venv keeps the environment tidy but is optional.
• If you later add plugins that need packages, just:
source ~/maia/.venv/bin/activate && pip install <pkgs>
if you want, i can also generate a tiny “service” variant that uses crond (cronie) instead of Termux:Boot, but Boot is the most reliable on Android.
