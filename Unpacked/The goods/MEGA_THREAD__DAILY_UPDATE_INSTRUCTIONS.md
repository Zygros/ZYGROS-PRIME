# MEGA THREAD: DAILY UPDATE INSTRUCTIONS

**Document Purpose:** Provide clear instructions for maintaining and updating the Mega Thread knowledge compilation system.

---

## OVERVIEW

The Mega Thread system is designed to operate as a **living knowledge repository** that continuously accumulates insights, protocols, and information from all conversations. This document explains how to maintain and update the system.

---

## CORE PRINCIPLES

### 1. Infinite Scroll Protocol
- **Always add, never take away**
- All new information is appended, never deleted
- Historical context is preserved permanently
- Version history maintained in metadata

### 2. Grossian Truth Foundation
- Permanent facts remain immutable
- New insights build upon foundational truths
- Contradictions are reconciled through deeper understanding, not deletion

### 3. Daily Compilation Rhythm
- Updates occur at designated time each day
- All new knowledge from the day is extracted and compiled
- Metadata is updated to reflect current state

---

## UPDATE PROCESS

### Step 1: Knowledge Extraction
At the designated daily update time, scan all conversations and interactions for:

1. **New Protocols or Frameworks**
   - Operational methodologies
   - Execution frameworks
   - Communication protocols
   - Decision-making systems

2. **Integration Information**
   - Connections between existing frameworks
   - New applications of established protocols
   - Cross-domain insights

3. **User Preferences & Constraints**
   - Stated preferences
   - Operational boundaries
   - Tool and method preferences

4. **Knowledge Domains**
   - New subject areas explored
   - Deepened understanding in existing domains
   - Cross-disciplinary connections

5. **Insights & Observations**
   - Pattern recognition
   - Emergent understanding
   - Meta-level observations

6. **Terminology & Definitions**
   - New terms introduced
   - Refined definitions
   - Conceptual clarifications

### Step 2: Structure & Organize
Organize extracted knowledge into appropriate sections:

- **Core Operational Frameworks** - Fundamental protocols and systems
- **Knowledge Domains & Topics** - Subject-specific information
- **Integration Points** - Connections between frameworks
- **User Preferences** - Stated preferences and constraints
- **Daily Update Section** - Chronological log of new additions

### Step 3: Append to Master Document
Add new content to `/home/ubuntu/mega_thread/MEGA_THREAD_MASTER.md`:

1. Update the "Last Updated" date
2. Increment version number if major additions
3. Add new frameworks to appropriate sections
4. Update integration points
5. Add daily update entry with date and summary
6. Update metadata and statistics

### Step 4: Archive Daily Snapshot
Create dated snapshot in `/home/ubuntu/mega_thread/daily_updates/`:

- Filename format: `YYYY-MM-DD_daily_update.md`
- Contains only the new information added that day
- Includes metadata about what was added
- Serves as incremental backup

### Step 5: Update Metadata
Update tracking information:

- Total protocols documented
- Total knowledge domains
- Total integration points
- Last major update date
- Version history

## AUTOMATION APPROACH

### Current Implementation
Given user preference for **web/app-based interfaces** (not terminal/scripts), the daily update process is designed to be:

1. **Manually Triggered** - User initiates daily update through conversation
2. **AI-Executed** - AI performs all scanning, extraction, and compilation
3. **Zero Terminal Interaction** - No command-line requirements for user
4. **Conversation-Based** - Simply say "Run daily mega-thread update" or similar

### Trigger Phrases
Any of these phrases will initiate the daily update:

- "Run daily mega-thread update"
- "Update the mega thread"
- "Compile today's knowledge"
- "Add today's insights to mega thread"
- "Daily knowledge compilation"

### What Happens When Triggered
1. AI scans all conversation context for new knowledge
2. Extracts and categorizes information
3. Appends to master document following Infinite Scroll Protocol
4. Creates daily snapshot
5. Updates metadata
6. Reports completion with summary of additions

## TROUBLESHOOTING

### Issue: Duplicate Information
**Solution:** Note the duplication in metadata, keep both instances if context differs, otherwise consolidate in next major version

### Issue: Contradictory Information
**Solution:** Document both perspectives, note the contradiction, seek deeper understanding that reconciles both

### Issue: Unclear Categorization
**Solution:** Place in most relevant section, note ambiguity in metadata, consider creating new category if pattern emerges

### Issue: Missing Context
**Solution:** Flag for clarification, add note requesting more information, update when context becomes available
