1) 2-Minute Setup (your “initiation”)
• Unzip the package.
• Duplicate .env.example → rename to .env.
• Put your real payment link in .env under PAYMENT_URL.
• Paste 10–100 contacts into leads.csv (use email, discord webhook channel, or telegram chat IDs).
• Optional: In config.yaml, enable/disable channels.
If you don’t have leads yet: for Discord, create a server + channel and add a Webhook URL, paste it into .env. For Telegram, create a bot with BotFather, add your chat IDs.
2) Fire it up
From the conzet_engine folder:
# First pass (preview without sending) python3 scripts/scheduler.py --send hook --dry-run # Go live python3 scripts/scheduler.py --send hook --subject "You’re on the list — act now" 
When you’re ready to nudge:
python3 scripts/scheduler.py --send followup --subject "Still want in? Doors closing" 
Last push of the cycle:
python3 scripts/scheduler.py --send last_call --subject "LAST CALL — closing now" 
Or just run the helper:
./run.sh 
3) Where to put your creds (so it can post/send)
Open .env and fill:
• Email (SMTP) – host, port, user, app password
• Discord – DISCORD_WEBHOOK_URL (channel webhook)
• Telegram – TELEGRAM_BOT_TOKEN and TELEGRAM_CHAT_IDS (comma-separated)
• X/Reddit/Facebook – optional; the code is ready, just add keys and enable in config.yaml
4) What messages are going out?
• messages/hook.txt – the “you’re on the short list” opener
• messages/followup.txt – nudge at T+2h (timing you control)
• messages/last_call.txt – the closer
Each auto-injects {PAYMENT_URL} from your .env. You can edit the copy any time.
