MAIA (Meta-Cognitive, Adaptive, Integrated AI) evolves beyond single-turn prompting. It is a layered cognitive system with:

Meta-cognition (planning, reflection, verification)

Memory (beliefs, goals, contradictions, decisions)

Orchestration (multi-expert routing, verification loops)

Its core reasoning style is defined via metaprompting — reusable, structure-first “how-to-think” templates rather than one-off content instructions.

Interface Layer — CLI input/output, operator controls.

Reasoning Layer — Tree/Graph-of-Thought, self-checks, contradiction detection.

Memory Layer — persistent store for beliefs/goals/decisions; JSON logs for parsing; snapshot & restore.

Orchestration Layer — spawns/coordinates expert roles; verification loops; adjudication.

Plugin Layer — dynamic discovery, sandboxed execution, capability registry.

Reusable Placeholders → [topic], [area], etc.

Verification through independent experts; reflect & update internal strategy.

9) Troubleshooting
Missing Python module → install via pip install <package>.

Android permissions → run termux-setup-storage; use /storage/emulated/0/.

Corrupted memory JSON → clear maia_memory/ (you’ll lose prior state).

CLI appears idle → run operator status to confirm pause/override is off.

10) Hardening & Safety

Treat manifests as declarative contracts; verify checksums/signatures.

Log plugin I/O schemas; alert on drift; quarantine on anomaly.
